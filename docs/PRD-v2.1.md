# PRD-002 记账 app V2.1 优化

## 背景
基于 v2 MVP 已发布，发现以下核心体验问题需要优化：
- 首页太简陋，信息密度低
- 添加账单入口深（要到账单页）
- 记账流程慢（分类选择、金额输入）
- 搜索筛选功能缺失

## 原始需求来源
- 飞书文档：https://www.feishu.cn/docx/AgX6dQoJYocQU9xsVvlc4qZFnJf
- 群内讨论（2026-04-29）

## 功能需求（V2.1）

### FR-11 首页优化
- **今日概览卡片化**：支出(红色)/收入(绿色)/结余，分开独立显示，数字更大更醒目
- **悬浮快捷按钮**：右下角悬浮「+记一笔」按钮，点击直接弹出记账表单（无需跳转）
- **首页查看全部入口**：底部加「查看全部」链接，跳转账单列表

### FR-12 记账流程优化
- **最近使用分类**：顶部常驻 3 个最近用过的分类，点击直接选中
- **智能记忆**：记住上次使用的「分类+类型」组合，首次打开自动填充
- **金额快捷按钮**：增加「×100」「×1000」按钮，方便输入大额（3万→3→×10000）
- **记账成功反馈**：弹出 toast 同时显示「本月已花 XX元，还剩 XX元」

### FR-13 搜索与筛选（P0）
- **关键词搜索**：按备注内容搜索
- **时间筛选**：Tab 切换「今天/本周/本月/全部」
- **分类筛选**：多选分类过滤
- **搜索结果**：列表展示，支持删除

## V2.2 规划（待确认后开发）

### FR-21 账户余额自动计算
- 根据账单记录自动汇总各账户余额，不再手动填写

### FR-22 报表增强
- 日报增加近 7 天趋势折线图
- 月报增加实际支出 vs 预算对比
- 饼图点击下钻到分类明细

### FR-23 周期账单
- 每月固定缴费（房租/订阅）自动生成记录

## V2.3 规划

### FR-31 分类预算
- 按分类设置月度预算（餐饮2000/交通500）
- 分类维度提醒

### FR-32 数据导出
- 导出 Excel/CSV
- 本地定期自动备份

### FR-33 微信登录
- 一键登录，无需密码

## 用户故事
- 作为用户，我希望在首页一眼看到今日收支概况，不需要点击就能了解当天财务状况
- 作为用户，我希望 3 秒内记完一笔账，不需要跳转页面
- 作为用户，我希望快速找到以前的某笔记录，支持关键词和时间筛选
